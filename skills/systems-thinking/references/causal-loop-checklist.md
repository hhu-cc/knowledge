# Causal Loop Checklist

Use this checklist before declaring a root cause.

## Variables

- What variables are changing over time?
- Which variables matter most to the outcome?
- Which variables are signals versus true drivers?

## Interactions

- How does each key variable affect the others?
- Which effects are immediate and which are delayed?
- Which relationships are strong, weak, or uncertain?

## Reinforcing loops

Look for cycles where change accelerates itself.
Examples:

- better outcomes increase trust, which increases adoption, which creates even better outcomes
- more delays create more firefighting, which creates even more delays

## Balancing loops

Look for cycles that resist change or stabilize the system.
Examples:

- demand rises, capacity is consumed, service quality drops, demand growth slows
- more governance reduces variance but also slows execution

## Delays

- How long after an action does the effect appear?
- Could a delayed effect make a bad intervention look good in the short term?

## Boundary checks

- What is outside this model but still relevant?
- Are we ignoring incentives, policy, trust, or coordination costs?
